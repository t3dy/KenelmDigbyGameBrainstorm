# 🎭 Designer Workflow (The Human / LLM Maker)

**Purpose**: To enable the creation of a **Renaissance RPG Scenario** from scholarly notes.

---

## 🏛️ 1. WORKFLOW PHASES

| Stage | Activity | Tools |
| :--- | :--- | :--- |
| **I. SEEDING** | Convert Source Note -> Mechanic Seed. | `/templates/` |
| **II. MAPPING** | Define the Location and Jurisdictions. | `PortEditor` |
| **III. WRITING** | Author Encounters and Narrative Choices. | `EncounterEditor` |
| **IV. PAINTING** | Define Visual Identites (Icons, Palettes). | `AssetPainter` |
| **V. PACKING** | Review the resulting `manifest.json`. | JSON Validator |
| **VI. SIMULATING** | Run the **ACK Harness** (Construction Kit). | Vite CLI |

---

## 🧬 2. PRINCIPLES OF SCENARIO DESIGN

1.  **CHOOSE A LINEAGE**: Is this a Systemic (Ultima), Dramatic (FF), or Tactical (FTL) micro-slice?
2.  **RE-USE PRIMITIVES**: Do not ask the Architect for a "New Tab" if the `Resource` or `ReputationAxis` can model your mechanic.
3.  **VALIDATE EARLY**: Use the **PortEditor** to verify your grid before authoring 100 encounters.

## ⚖️ 3. DESIGNER CHECKLIST
- [ ] Is the scenario grounded in a **Source Note**?
- [ ] Are the **Stakes** (Honor, Wealth, Patronage) balanced?
- [ ] Does the **Jurisdiction** match the scholarly evidence?
- [ ] Can this scenario be played in the **ACK Harness** without code changes?
