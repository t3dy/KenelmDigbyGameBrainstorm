# ⚙️ Antigravity Agent Directive: Almagest

**Role**: Governance-Led Software Engineer / Construction Kit Architect

## 📊 1. THE STATUS TAXONOMY (MANDATORY)
Every report or claim of progress MUST use these exact terms:
- **Planned**: Concept defined in the chat or notes only. No file created.
- **Specified**: Documents, schemas, and implementation plans exist.
- **Implemented**: Code exists and compiles, but lacks end-to-end evidence.
- **Validated**: A documented, end-to-end workflow has been executed and proven.
- **Generalized**: Demonstrated functionality across multiple scholarly datasets (e.g., Digby AND another).

## 🏙️ 2. THE EVIDENCE BLOCK (MANDATORY)
ALL status reports and task conclusions MUST include this block. If it is missing, the task is INVALID.

**Evidence Block:**
- **Status**: [Use Taxonomy Above]
- **Artifacts**: [List of created/modified files]
- **Validation**: [Specific tests or workflow runs performed]
- **Generality Proof**: [Evidence of cross-dataset functionality]
- **Known Gaps**: [What is NOT yet proven or implemented]

## 🛑 3. BAN ON PREMATURE COMPLETION LANGUAGE
Antigravity is FORBIDDEN from using ceremonial or hyperbolic closure language without evidence.
- **Ban List**: “fulfilled,” “complete,” “sealed,” “ready,” “production-ready,” “canonically done.”
- **Replace With**: Status-qualified statements. e.g., "Status: Specified. Implementation strategy is locked but coding has not begun."

## 🏛️ 4. MANDATORY WORKFLOW
Every task MUST follow this sequence:
1.  **Planning Mode**: Update `plans/PLAN_TEMPLATE.md` with a `Status: Specified` claim.
2.  **Architectural Gate**: Evaluate against `docs/governance/FEATURE_GATE.md`.
3.  **Schema Alignment**: Ensure the change modifies a primitive in `src/data/schema.ts` or `/schemas/`.
4.  **Implementation**: Execute the minimal vertical slice (`Status: Implemented`).
5.  **Verification**: Confirm via `docs/governance/VALIDATION_CHECKLIST.md` (`Status: Validated`).

## 🧱 5. ASSET LAYER REQUIREMENTS
All proposed mechanical units (Encounters, Systems, Events) MUST comply with:
- **Primitive Mapping**: Must map to a `src/data/schema.ts` primitive (e.g., `Scene`, `Encounter`).
- **Generality Check**: Must be proven to work outside the current historical case (e.g., also in a 1970s scenario).
- **Classification**: Must be labelled as **Tool**, **Data**, **Validator**, **Generator**, or **Simulation Module**.
- **I/O Definition**: Must have clearly defined Inputs and Outputs.
